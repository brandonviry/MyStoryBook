# CarrerAnimate360 - Guide Pedagogique

## 1. Analyse pas a pas

1. React rend un bloc simple.
2. Framer Motion applique une rotation via `animate`.
3. La transition controle la duree.

## 2. Concepts cles (mots simples)

Pense a une roue:
- React cree la roue.
- Framer Motion la fait tourner.

## 3. Raisonnement React (mount / update / unmount)

- **Mount**: le bloc est monte dans le DOM.
- **Update**: si les props changent, React re-render.
- **Unmount**: le bloc est retire du DOM.

## 4. Raisonnement Framer Motion

- `animate` decrit l'etat final (ici: rotation).
- `transition` decrit la vitesse (ici: duree).

## 5. Erreurs et mauvaises pratiques

- Oublier la taille ou la couleur -> on ne voit pas l'animation.
- Duree trop longue -> effet mou, peu lisible.

## 6. Version corrigee et optimisee (simple)

```tsx
import { motion } from "motion/react";

export default function Example() {
  return (
    <motion.div
      style={{ width: 100, height: 100, background: "#1d4ed8" }}
      animate={{ rotate: 360 }}
      transition={{ duration: 1.2 }}
    />
  );
}
```

## 7. Exemple minimal fonctionnel

```tsx
<motion.div animate={{ rotate: 360 }} transition={{ duration: 1 }} />
```

## 8. Resume a retenir

- React rend le bloc, Motion anime la rotation.
- `animate` = etat cible.
- `transition` = timing.
